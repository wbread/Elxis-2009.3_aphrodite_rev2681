Elxis Team suggestions for your site

Things you should avoid in your site

  • Do not install Elxis CMS in a subdirectory.
  • Do not create ugly sites! A web site is a mirror of ourselves, or of our company. Spend a little time or money to have the best site possible.
  • Do not create an insecure site. Check PHP settings, Elxis CMS settings and your file system. If you are unsure hire a professional.
  • Do not install incompatible with Elxis software.
  • Do not use modules that you really don't need and most probably none of your visitors use. (i.e. No one visits your site to see what time it is!).
  • Do not set PHP in Safe Mode.
  • Do not allow user registration except if you really need it.
  • Avoid too many javascript and flash in Front-End.
  • Do not use other character encoding than UTF-8.
    Elxis is optimized for UTF-8 and you can display any language with UTF-8.
  • Do not use templates with massive PHP code.
    A template should be almost only html without PHP logic. Get XHTML/CSS valid templates, optimized for search engines.
  • Avoid using Elxis default template, it is just a sample!
  • Avoid the use of the PDF generation feature. It is too CPU consuming.
  • Avoid tables and complicated html in your articles.
  • Do not use HTML to format text. Use CSS instead.

Things you should do

  • Find a good hosting company with good support and secure environment.
    Don't go to the cheapest one. Imagine what will be the trouble and cost if you loose your site. Search Elxis.org website for Elxis official authorized hosting providers.
  • If you are not familiar with site creation, CMSs, web, templates, editors, etc., hire professionals. It will cost you some money but it will save you valuable time that you can spend somewhere else. At the end you will have a great site that will be an asset for you and your company. Search Elxis.org for official authorized solutions providers for Elxis CMS.
  • Take a database backup regularly using the Elxis CMS backup tool, or other external application.
  • Use Elxis CMS "SEO PRO" option. Select good SEO titles related to content.
  • Add META keywords and META description everywhere. Don't add too many keywords and long description.
  • Use as less Bots as possible and unpublish the ones that you don't need.
  • Use static cache (and gzip compression) on heavy loaded web sites.
  • Optimize/Vacuum your database regulary.
  • Hide administration login page and/or enable Logins Recorder tool (inspect logs regulary).
  • Use FTP access over files. For maximum security disable FTP access when you don't need it and re-enable it before you need to work with files.
  • Use xhtml, css valid templates. Validators:
    http://validator.w3.org/,
    http://jigsaw.w3.org/css-validator/,
    http://validator.w3.org/checklink
  • In your articles use as less html as possible.
    Use html elements that have some meaning like h1, h2 and p.
  • Your site should give emphasis to content (text).
  • Update your site when an update is available from elxis.org.
  • Update third party software when an update is available.
  • Use a Sitemap component and CSS based menus for search engines optimization.
  • Get professional support by Elxis CMS official authorized support companies for your site if you want to be sure that anytime, any problem, will be solved by professionals.
  • Download Elxis extensions from elxis.org or elxis-downloads.com official websites. Do not trust other sources. Make sure the extensions you downloded/purchased are compatible with Elxis and optimized for it before install.
  • Make needed folders and files (like /tmpr and /cache) writable by PHP.
  • Use Firefox web browser to administer your site.
