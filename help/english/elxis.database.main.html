Database Manager

The database manager component, is a very useful component for administrators, as it helps them to view important information about Elxis database from Back-End without needing an extra application, like phpMyAdmin.

Administrators can:

  • View database structure.
  • View tables structure.
  • Get information for the database.
  • Dump database structure and data in SQL format.
  • Dump database structure and data in XML format.
  • Download database dumps or save them on server.
  • Monitor Stats
  • Monitor Tables
  • Optimize database
  • Repair database

All files are UTF-8 Encoded.

 

Toolbar Icons

Manage Backups: Opens the 'Database Manager: Backups' page, described in another help screen, from where you create new database backups or delete older.

Help: This Help Screen.

 

Page is divided in two areas. The left where you can see existing tables and the right from where you can perform operations on the database.

 

Left Area

The left area shows all the database tables. Each row has two icons. The "Browse" and "Structure".

Click the "Browse" icon to view table's data. Below the data table, there are two links. Click "Table Structure" to view a detailed view of  table's structure. Click "Back to Database" to return to this screen.

Clicking the table name or the "Structure" icon, you can view a detailed view of table's structure. Below table's structure table, there are two links. Click "Browse table" to browse table's data. Click "Back to Database" to return to this screen.

 

Right Area

The right area has three tabs: "Information", "Dump Database" and "Maintenance".

 

Information tab

The information tab shows information for the database.

Dump Database tab

Using the controls on this tab, you can backup database data and structure in XML or SQL format.

How to backup in XML format
  • Set 'Backup Type' to XML.
  • Select type of backup.
    Check "Structure & Data" to create a backup of structure and data. Check "Structure Only" to create a backup only for the structure of the database.
  • Select destination
    Check "Save on Server" to create a local backup file that you can download later. You can manage these backups from 'Database Manager: Backups'.
    Check "Download" to immediately download the backup to your computer.
  • Press "XML Backup" button to start the backup.

 

How to backup in SQL format:
  • Set Backup Type to SQL.
  • Select type of backup
    Check "Structure & Data" to create a backup of structure and data. Check "Only Structure" to create a backup only for the structure of the database. Check "Only Data" to create a backup only for the data.
  • Set Options:
    Lock Table: Select "Yes" to lock the table during backup. Useful if the database contains tables full of data.
    Full Syntax: Select "Yes" to create full mode INSERT SQL statements. Recommended.
    Drop Tables: Select "Yes" to add to backup file commands, to drop excisting tables during restore.
    Tables: Select the tables you want to backup. Checking "All Tables" you backup all tables. Checking "Selected tables", appears a dropdown list of the available tables. All tables are selected by default. You can select the tables you want, having the CTRL key pressed.
  • Select destination
    Check "Save on Server" to create a local backup file that you can download later. You can manage these backups from 'Database Manager: Backups'.
    Check "Download" to immediately download the backup to your computer.

 

Maintenance tab

You can use these two functions to optimize and repair the database used by Elxis.

Optimize: Optimizes database and increases performance.

Repairs: Repairs any existing errors in database tables.
