Glossaire Elxis

Ce glossaire, vous résume les termes les plus important pour le CMS Elxis.

Access Manager (Gestion des Accès): A système intégré et flexible pour les administrateurs pour définir les accès des groupes, pour appliquer les droits et contrôle des accès.

Admin Templates (Template Administration): Défini la charte graphique de l'Administration de Elxis.

Administrator (Administrateur): Ce sont les utilisateurs et responsables du site via l'interface d'administration ils sont, alors en mesure de créer et de maintenir le site. Le site Elxis peuvent avoir plus d'administrateur.

Administrator (Super) (Super Administrateur): Il est la plus haute autorité dans la hierarchie. Il est responsible pour tout. Le Super administrateur ne peut pas être supprimé.

Alt Tag (Alt tag): L' attribut "alt" , indépendement de "alt tag" est un texte attribué et ajouté à une image pour la décrire. Ces texte est visible (derrière une boîte de dialogue) quand le visiteur survole l'image sur la partie public du site.

Archive (Archives): Ce sont les éléments de contenu, articles qui sont archivés par l'administrateur. Le Module d'Archive fournit une façon d'afficher des articles archivés sur votre site Web.

Automatic Translations (Traduction Automatique): Elxis peut vous aider à traduire le contenu d'un langage à un autre. Inclut un mécanisme de traductions automatiques qui dans les secondes créent un nouvel article ou une page autonome, basé sur la langue initiale. S'applique pour aux articles et aux pages autonomes.

Autonomous Page: (Pages Autonomes) Un article qui n'appartient ni à une catégorie ni à une section. Autonome est un mot grec pour & quot;Independant".

Back-End (Administration): C'est le panneau de contrôle de l'"Administration" de Elxis, avec lequel les administrateurs du site gérent le site et son contenu.

Blog (Blog): Un style de Blog pour Elxis est la liste d'Articles dans un formulaire sommaire. Le texte d'intro est utilisé pour créer le blogs. Cliquez sur le lien "En savoir plus..." vous affichera l'article complèt.

Bot (Bot): A Bot est une petite fonction, tâche-orientée qui intercepte le contenu avant d' être affiché et le manipule d'une certaine façon. Elxis fournit plusieurs Bots dans sa distribution. Le nom de bot est un héritage de Mambo où il a été introduit comme mambot.

Category (Catégorie): La catégorie est le nom pour un ensemble d'articles qui sont liés d'une certaine façon. Les catégories appartiennent aux Sections. Exemple: La catégorie "Villes" pourra contenir des enregistrements comme "Athènes", "Londres" et "Rome".

Check Out (an item) (Vérifier (un article)): Quand un utilisateur modifie un fichier il est en position "Vérifié". Cela empêche les autres utilisateurs d'être capable de modifier l'article en même temps, et donc de prévenir la perte de données au moment de l'enregistrement de l'article.

CMT (CMT): CMT est un acronyme pour Composants, Modules et Templates.

CMTB (CMTB): CMTB est un acronyme pour Composants, Modules, Templates et Bots.

Components (Composants): Les composants de Elxis sont des éléments ou applications qui habituellement aparaissent dans le centre du site. Les composants sont des éléments de fonctionnalités de Elxis. Elxis inclue les composants suivants: Bannières, contact, gestionnaire de base de données, Flux de nouvelles, sondages et Liens Web. Vous pouvez installer de nouveaux composants. Pour ce faire visiter les sites www.elxis.org ou www.goup.gr pour plus d'information.

configuration.php (configuration.php): C'est le fichier des paramètres ou la configuration générales du site sont stockés. Pour les raisons de protection celui-ci devrait être a non-modifiable en temps normal.

Content Item (Elements de Contenu): Les éléments de contenu sont des "articles" enregistrés dans une catégorie.

Content Management System (CMS): Un CMS est une sorte de logiciel de gestion de contenu pour un site Web à partir d'un serveur.

Core (Noyau): Le noyau Elxis est un fichier compressé qui contient l'installation standard et officielle de Elxis, lequel peut être trouvé sur Elxis.org et GO UP Inc. . Il contient quelque CMTM pré-installé, mais peut être étendu avec des add-ons créés par des développeurs tiers.

Database (Base de Données): Une base de données est une collection d'informations. Elle contient un ensemble d'enregistrements que vous pouvez chercher, trier et analyser rapidement. Elxis supporte les bases de données de type MySQL et Postgres

Extensions (Extensions): Les extensions sont des add-ons utilisées par EDir et ajoute des fonctions supplémentaires (exp. reservations, boutique en ligne, téléchargements, galeries, etc.

Front-End (Partie-Public): C'est la partie visible du site web que les visiteurs verront. Vous administrez la Partie-Public à partir de l'administration.

Frontpage (Page d'Accueil): La page d'accueil est un Composant qui montre tous les articles publiés sur votre site, marqué avec le paramètre & quot;Affiché en page accueil". Vous pouvez configurer l'affichage du la Page d'Accueil en éditant l'article du menu associé avec lui, habituellement 'Accueil'.

Global Checkin (A Vérifier): Quand un article est "A vérifiert" c'est qu'il n'a pas été sauvegardé correctement ou mal annulé (exp. en cas de déconnexion), il sera remis de sa position "A vérifier", par un administrateur et sera de nouveau disponible.

Global Configuration (Configuration Globale): PLes paramètres sont enregistré dans par la configuration global pour le comportement de votre site Elxis. Ils sont très importants et devraient être traités avec soins. Les paramètres de configuration globaux sont stocké dans le fichier configuration.php.

Intro Text (Texte d'Intro): Quand vous créez ou éditez un Article, le Texte Intro peut contenir une brève introduction. Le Texte Intro est utilisé dans le mode du blog, suivi par un lien "Lire la Suite..." qui servira à lire l'article en entier (Intro et corps de texte).

Language (Site) (Langue (Site)): C'est la langue utilisée sur la partie-public. Les utilisateurs peuvent changer de langue et affichez le contenu qui est fourni pour cette langue. Vous pouvez installer de nouvelles langues sur votre site.

Language (Administrator) (Langue (Administration)): C'est la langue utilisée sur la partie administration. Vous pouvez changer la langue de l'interface d'administration. Vous pouvez installer de nouvelles langues pour l'administration.

Language (Translation) (Langues (Traduction)) : Si vous le souhaitez, vous pouvez traduire Elxis dans votre langue et nous envoyer les fichiers traduits à partir de www.elxis.org. Veuillez visiter www.elxis.org pour de plus amples informations.

Main Text (Corps du Texte): Quand vous créez ou éditez un Article, votre article contient le corps preincipal de votre article. Vous pouvez écrire des articles qui utilisent seulement le corps ou un texte d'intro. quand vous voulez afficher les articles dans le blog , vous devez cliquer sur le lien "Lire la suite..." pour voir l'article en entier (Intro et Corps du Texte).

Elxis (Elxis): Elxis est système de gestion de contenus (CMS) multilangues.

Elxis Administration (Administration Elxis): L'administration de Elxis est une partie du CMS de Elxis et vous permet d'administrer Elxis pour votre site web. Quelquefois il est fait référence aussi à Admin, Administration, Panneau d'Administration ou Panneau de Contrôle.

Elxis Defender (Elxis Protection): Des outils vous aidant à protéger votre site des attaques.

Elxis Team (Equipe de Elxis): Des gens qui contribuent au développement de Elxis.

Elxis Community (Communauté Elxis): Des gens qui participent via le forum de Elxis.

Floodblocker (FloodBlocker): Un outil qui vous aide à protéger votre site des attaques.

Media Files (Fichiers Média): ce sont des fichiers avec l'extension .gif, .jpg, .png or .bmp, doc, mpg, xls, avi, flv, pdf, ppt, etc, ces fichiers seront utilisés dans les Contenus et sont organisés via le gestionnaire de Media.

Member (Membres): Voir "Utilisateur Enregistré".

Menu (Menu): Utilisé pour les buts de la navigation c'est une caractéristique principale d'Elxis. Les menus consistent en des liens d'un ou plus d'articles qui peuvent être organisés hiérarchiquement. Les menus sont affichés dans les modules utilisés sur la partie-public. Vous pouvez avoir autant de menus que vous vous voulez.

Menu Item (Elements de Menu): Les éléments de menu sont des liens qui forment des Menus. Elxis supporte plusieurs types d'élémenes de menu. Chaque type d'élément de menu est utilisé pour afficher un contenu ou d'autres choses pour une direction particulière.

Modules (Modules): Les modules étendent les capacités de Elxis, en donnant de nouvelles fonctionnalités au logiciel. Les modules sont de petites applications qui peuvent être affichées à l'intérieur de positions du module définies dans un template. Les modules inclus dans Elxis sont: Menu Principal, Menu Top, Choix de Template, Sondages, Flash, Compteur de Visite, etc. Vous pouvez installer d'autres modules. Visitez www.elxis.org ou www.goup.gr pour de plus amples informations.

Multilingual (Multi-Langues): Multi-langue est pour les sites web qui peuvent afficher leur contenu dans plusieurs langues. Avec Elxis vous pouvez créer des sites web multi-langues avancés, vous pouvez même avoir des dispositions différentes ainsi que leur contenu la structure est basée sur la langue sélectionnée.

Newsfeeds (Flux de Nouvelles): Flux de Nouvelles est une façon de partager le contenu entre des sites différents. Nous appelons Flux de Nouvelles les articles qui sont publiés dans d'autres sites web et les propriétaires du site nous permettent de les afficher sur le votre à travers les Flux RSS. Flux de Nouvelles n'affiche pas l'article en entier, mais seulement le titre et une courte introduction. Si vous trouvez l' article intéressant, vous pouvez cliquer sur le titre et lire l'article dans le site qui l'a publié. Elxis peut afficher des Flux de Nouvelles et fournir des Flux RSS pour d'autres sites.

Open Source (Open Source): Open Source est un terme géneral qui decrie les différentes parties du monde des Licences Logiciels Libres. "Logiciels Libres" est une manière de liberté, sans prix. Pour comprendre le concept, vous devriez penser que "libre" est une "liberté d'expression", mais pas une "bière gratuite".

Parameters (Paramètres): Les paramètres peuvent être utilisés pour contrôler la présentation, ou la position des articles, menu, composants, bots ou modules.

Preview (Aperçu): Vous pouvez afficher un aperçu de votre site dans l'administration (comment les visiteurs verront le site) des contenus et des modules.

Publish (Publier): Pour qu'un article soit visible sur la partie-public il devra être publié. Seuls les Articles publiés seront visibles. Veuillez noter que dans certains cas plus de critères doivent être informés pour q'un article soit visible. Les plus important d'entres-eux sont: Niveau d'accès et langue.

Record (Enregistrement): Un Enregistrement est un terme générique pour un "ligne" dans une base de données. Un enregistrement représente très souvent une "piece" d'un contenu. Les fonctionnalités dynamiques et le contenu d'Elxis repose sur une base de données pour fonctionner.

Registered User (Utilisateur Enregistré): Un utilisateur qui a un compte sur votre site Elxis peut se connecter et accéder au contenu adressé seulement aux membres. Appelé aussi membre.

Section (Section): Une Section est un ensemble de Catégories qui sont liées d'une certaine façon. Pensez dans cette direction. Par exemple une section pourra être appelé "Vins". Les Catégories de cette section seront "Blanc", "Rouge", "Rosé", "Pétillant", etc.

SEO title (Titre SEO): Voir "Titre (SEO)".

Site (Site Web): Le Site Elxis est le site web affiché pour vos visiteurs ou utilisateurs, il contiendra tous les contenus ajoutés dans par les administrateurs. Il est fait référence aussi à la "Partie-Public".

SoftDisk (SoftDisk): SoftDisk est une zone de la mémoire virtuelle pour les variables et paramètres en tout genre. Vous pouvez ajouter ou modifier/supprimer des entrées existantes dans SoftDisk sauf celles étiquetées comme utilisées par le système. Aussi les entrées marquées à "Ne pas supprimer" peuvent seulement être éditées. Pour nommer les sections de SoftDisk et les noms des valeurs vous êtes seulement autorisés à utiliser des lettres en latin, numériques et le symbole (_).

Submitted Content (Soumettre un contenu): C'est le contenu (nouvelles, articles, etc) qui est soumis par les utilisateurs de la Partie-Public. Les administrateurs, inspecteront le contenu soumis et décideront de le publier ou pas.

Super Administrator (Super Administrateur): Voir "Administrator (Super)".

Templates (Administrator) (Templates Administrateur): Les Templates d'administration définissent le look de l'administration de Elxis (Administration). Vous pouvez installer de nouvelles templates.

Templates (Login Screens) (Templates Ecran de Connexion): Les Écrans de connexion sont des modèles qui contrôlent ce qu'un utilisateur voit quand il essaie de se connecter à l'Adminsitration. Vous pouvez installer de nouveaux écrans de connexion.

Templates (Site) (Templates (Site)): Les Templates du sites définissent la charte graphique de votre site Elxis (Partie-Public). Vous pouvez assigner différentes templates à chaques pages du site. Vous pouvez installer de nouvelles templates.

Title (Titre): C'est le nom affiché pour les articles.

Title (SEO) (Titre (SEO)): C'est un texte qui sera utilisé pour former l'URL d'une partie de Elxis quand SEO Pro sera installé. Ce texte peut être défini par administrateur ou proposé par Elxis.

Tools (Outils): Les outils sont des applications disponibles aux administrateurs qui seront utilisés généralement pour l'entretien du site. Les outils sont une autre caractéristique unique du CMS Elxis.

User (Utilisateurs): Un terme général qui décrit les personnes qui ont créé un compte sur le site Elxis et qui appartiennent à un groupe utilisateur. Voir aussi "Visiteurs" et "Utilisateurs Enregistrés".

Wrapper (Encapsulateur): Un Encapsulateur est un composant Elxis dans lequel vous pouvez afficher une page URL dans votre site Web. Cela peut être utilisé pour mettre une application comme un forum ou une galerie, des pages individuelles aussi bien que un site Web entier. Son nom alternatif est IFrame.
