Information du Système

L'information est affichée sous forme d'onglets comprenant trois sections principales.

Onglet système: C'est un aperçu général du serveur et des versions des composants actuellement installés sur votre site Elxis. Vous trouverez ici la version/build de votre PHP, la version de votre base de données MySQL ainsi que la version de Elxis et les paramètres importants qui ont été réglés par votre système et qui pourraient interférer avec votre installation actuelle. Le fichier de Configuration affiche le contenu de votre fichier configuration.php. Ce fichier est généré lorsque vous installez Elxis sur votre serveur, et il est modifé quand vous appliquez des changements dans la section configuration générale de l'administration de Elxis. Ce fichier définit plusieurs variables globales utilisées lors de l'exécution de Elxis. Vous ne pouvez pas éditer ce fichier ici mais cela vous permet de vérifier les paramètres actuels pour résoudre des problèmes ou pour copier l'information dans une demande de support

Onglet PHP Info: PHP est le langage de script côté serveur sur lequel repose Elxis. Ce langage est installé et exécuté sur le serveur et par conséquent tous les paramètres sont réglés sur ce serveur. Tout ce que vous avez toujours voulu savoir sur l'installation PHP de votre serveur est affiché ici. La possibilité que vous avez de changer cette information dépend de si le serveur vous appartient ou si celui qui l'héberge est ouvert à collaborer avec vous sur les changements nécessaires. L'onglet PHP vous permettra de voir tous les paramètres et variables qui définissent votre installation de PHP. C'est une bonne habitude de s'assurer de connaître quels sont les pré-requis nécessaires pour tous les composants 'Tiers' que vous prévoyez d'utiliser avec Elxis. Vous pouvez ensuite vérifier la présence de ces pré-requis ici.

Onglet Permissions: Affiche une liste des éléments, répertoires critiques de Elxis cela vous permet de voir ce qui est modifiable ou pas .
IMPORTANTE NOTE DE SECURITE: Dans des conditions normales d'urilisation, seuls les répertoires /tmpr et /cache doivent être accessible en écriture (Modifiable). TOUS LES AUTRES répertoires doivent être à NON-MODIFIABLE. Vous devez créer un autre répertoire modifiable à utiliser uniquement quand vous souhaitez transférer des données. Immédiatement après vous devez le mettre à non-modifiable.
Si vous avez ACTIVE LE FTP, il est possible de transférer des données même lorsque les répertoires ne sont pas modifiables.
