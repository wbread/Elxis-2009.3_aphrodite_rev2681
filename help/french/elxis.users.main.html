Gestion des Utilisateurs

Cette page affiche la liste des utilisateurs enregistrés pour votre site. Vous pouvez ajouter, éditer les paramètres des utilisateurs, supprimer les utilisateurs, ou forcer la déconnexion.

 

Icones de la barre d'outils

Nouveau: Cliquez sur l'icône "Nouveau" pour créer un nouvel utilisateur.

Editer: Editer un profil d'utilisateur.

Supprimer: Supprimer un utilisateur.

Forcer la déconnexion: Cliquez sur l'icône "Forcer la déconnexion" pour déconnecter un utilisateur.

Aide: Cet écran d'aide.

 

Colonnes

Veuillez noter que certaines colonnes ont un filtre associé entre elles.

Nom: C'est le nom de l'utilisateur.

État de Connexion: Montre si l'utilisateur est connecté ou pas.

Activé: Montre si l'utilisateur est autorisé à se connecter sur le site. Cliquez sur l'icône pour changer ce paramètre.

ID Utilisateur: C'est l'identifiant de l'utilisateur, de l'administrateur dans Elxis.

Groupe: C'est le groupe dont l'utilisateur est membre. En utilisant le Gestionnaire d'Accès, les administrateurs peuvent créer de nouveaux groupes. Vous pouvez utiliser le filtre relatif pour montrer seulement les utilisateurs qui appartiennent à un groupe spécifique. Utile pour les sites web avec un grand nombre d'utilisateurs enregistrés.

E-mail: C'est l'adresse e-mail de l'utilisateur.

Dernière Visite: C'est la date et l'heure à laquelle l'utilisateur a visité une page du site en étant connecté.

Contrôle des niveaux d'Accès

Les utilisateurs enregistrés du site appartiennent à plusieurs groupes. Ces groupes sont divisés en deux gategories principaux: Public et Serveur. Les utilisateurs qui appartiennent aux groupes Public ont seulement accès à la partie Public. Les utilisateurs qui appartiennent aux groupes Serveur ont accès aux deux parties: Public et administration.

Elxis est livré avec plusieurs groupes prédéfinis. Il donne aussi la liberté aux administrateurs pour définir leurs propres groupes d'utilisateurs.

Les groupes utilisateurs sont sous la direction de Elxis qui utilise pour mettre des priviliges et des droits d'accès aux utilisateurs. Basé sur eux, les utilisateurs peuvent afficher, avoir accès à l'info pour les utilisateurs enregistrés ou qui soumettent un contenu qui sera utilisé sur la partie-public ou l'administration.

 

Groupes Public

Enregistré - Ce groupe permet à l'utilisateur de se connecter à l'interface d'utilisateur.

Auteur - Ce groupe permet à l'utilisateur de poster un contenu, généralement par le lien dans le Menu Utilisateur.

Editeur - Ce groupe permet à l'utilisateur de poster et d'éditer les éléments depuis l'interface d'utilisateur.

Rédacteur - Ce groupe permet à l'utilisateur de poster, éditer et publier les éléments depuis l'interface d'utilisateur.

 

Groupes de section d'administration

Manager

Ce groupe permet d'accéder au contenu Création et Information Système

Menus d'administration accessibles:

  • Aide (général), incluant Information Système.
  • Site: Gestion Media , Aperçu.
  • Menu: Edition de menu existant, incluant création d'éléments de Menu. Toutes les fonctions en relation.
  • Contenu: Contenu, Contenu Statique, Section , Catégorie gestion, Page Accueil et Archives de gestion. Toutes les fonctions en relation.

Administrateur

Ce groupe permet l' accès aux principales fonctions d'administration.

outes les fonctions accessibles au groupe Manager, plus:

  • Site: Statistiques,gestion des rejets, gestion des utilisateurs (peut créer/éditer/supprimer tout type d'utilisateur excepté super administrateur)
  • Menu: Gestion des Menus
  • Modules, Components, Bots: Installer/Désinstaller, Créer, Editer. Exception pour le Composant de courrier de masse(mass mail).

Super Administrateur

Ce groupe permet l' accès à toutes les fonctions d'administration.

Toutes les fonctions accessibles au groupe gestion et à l'administrateur , plus:

  • Site: Configuration Globale, gestion des langues, gestion des templates, gestion des utilisateurs (peut créer/éditer/supprimer tout type d'utilisateur excepté les Super-Administrateur; Note 1:les Super-Administrateur peuvent être supprimés depuis la base de données; Note 2: les détails des Super-Administrateur peuvent êtres modifiés par un autre Super-Administrateur
  • Composants: Courrier de masse(mass mail).
  • Messages: Boite de réception, Configuration.
  • Système: Nettoyage du Cache (si le Cache à été permis dans la Configuration Globale), Enregistrement Global.
