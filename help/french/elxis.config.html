Configuration générale

La Configuration générale contrôle les paramètres de fonctionnement de Elxis. Les réglages effectués sur cet écran seront enregistrés dans le fichier configuration.php. Pour pouvoir modifier ces paramètres, il faut donc que ce fichier soit accessible en écriture.. Vous pouvez changer son état en Modifiable pour l'éditer et/ou le modifier. Ensuite ne pas oublier de le remettre à l'état de non-modifiable (644).

Remarquez que Elxis montre l'état actuel du fichier configuration.php. Il peut être soit "Modifiable" ou "Non Modifiable".

Cochez la case "Mettre à Non-Modifiable après sauvegarde" cocher la case pour le rendre Non-Modifiable après avoir enregistré le fichier. Mais la méthode la plus courante consiste à utiliser un logiciel FTP.

 

Les paramètres de configuration sont organisés en 11 groupes.

 

Onglet Site

Site hors-ligne - Ce réglage permet de rendre visible, ou bien de cacher, votre site ("public"). Utilisez cette fonction si vous mettez à jour des modules ou bien des articles, et que vous ne souhaitez pas que vos visiteurs voient le site tant qu'il n'est pas fini. Pensez à remettre le site "En ligne" lorsque vous avez terminé !

Message du site hors-ligne - Lorsque le réglage "Site hors-ligne" est en position Accès Indisponible, le message que vous saisissez dans cette zone sera affiché. Par défaut, il s'agit de "Ce site est en cours de modification. Merci de revenir nous voir un peu plus tard !". Vous pouvez le modifier comme vous le souhaitez.

Message d'erreur système - Ce message sera affiché si la connection initiale avec la base de données échoue. Vous pouvez saisir ce qu'il vous plait dans cette zone.

Nom du Site - Cette option vous permet de spécifier le nom du site. Il sera affiché dans le titre de la fenêtre de votre navigateur et sera utilisée lors des messages qui seront envoyés à partir de votre site.

Composants liés aux système d'Accès - TC'est un réglage très important qui est lié directement aux restrictionx d'accès. Elxis implémente un système de restriction d'accès avancé (Site - & gt; Accès au gestionnaire). Utiliser cette option, pour pouvoir définir où cette restriction d'accès sera appliquée.

  • Aucun: - Aucun contrôle d'Accès ne sera appliqué.
  • Aux composants du noyau: - La restriction d'accès sera appliquée seulement aux composants du noyau. Les composants du noyau sont ceux qui font tourner Elxis.
  • Aux composants tiers: - La restriction d'accès sera appliquée seulement aux composants que vous avez ajouté après l'installation de Elxis.
  • A tous les Composants: - La restriction d'accès sera appliquée à tous les composants (Noyau et Tiers).

Si vous mettez l'option à Aucun, vous désactivez les permissions de restriction d'accès. Si vous avez déjà installé des règles celles-ci resteront mais ne seront pas utilisés. Nous vous conseillons d'être très prudent avec option. C'est une des choses qui donnent à Elxis ses solutions unique et avancé.

Utilisation des Images de Sécurités - Mettre à Oui (recommandé) si vous souhaitez utiliser les images Captcha à faire apparaître dans les formulaires de Elxis. Les images Captcha peuvent vous aider à limiter les spams.

Voir Liens Non-Autorisés - Avec cette option activée, tous les visiteurs pourront voir le contenu auquel ils n'auraient pas droits autrement Accès (c'est, satisfaisant pour donner un Niveau d'accès 'Enregistré' ou 'Spécial'). Avec cette option désactivée, ils verront seulement le contenu avec un Niveau d'accès 'Public.' Le niveau d'accès pour le contenu est sélectionné sous la Publication et l'Édition de l'écran contenu.

Autoriser l'enregistrement des utilisateurs - Cette option vous permet de choisir pour tout le site si les visiteurs de votre site peuvent s'inscrire en tant qu'utilisateur enregistré. Si le bouton 'Oui' est choisi, le Module d'identification affichera le lien Aucun compte créé? Voulez vous en créer un. Si c'est le bouton 'Non', les visiteurs n'auront pas la possibilité de s'enregistrer.

Utiliser l'activation des comptes - Si le bouton 'Oui' est choisi, un lien d'activation sera envoyé à l'adresse e-mail donnée par l'utilisateur. Celui-ci ne pourra s'identifier qu'une fois qu'il aura cliqué sur le lien qu'il aura reçu. Si c'est le bouton 'Non' qui est choisi, un utilisateur pourra s'identifier immédiatement, sans passer par cette phase d'activation initiale de son compte.

Exiger un e-mail unique - Permet de choisir si vous voulez empécher l'enregistrement d'utilisateurs ayant la même adresse e-mail. Combinée avec l'option d'activation des comptes, cette fonction permet d'empécher un utilisateur de s'inscrire plusieurs fois, avec la même adresse e-mail.

Deboguer le site - Cette option vous permet de voir les messages d'erreur créés par votre installation de PHP. Cela peut être utile pour résoudre d'éventuels problèmes, ou pour décrire des erreurs à votre administrateur système ou à votre hébergeur. Le niveau de détail du rapport d'erreur peut être choisi dans le menu Configuration générale, sous l'onglet Serveur et peut être: Système par défaut, Aucun, Simple, Maximum

Editeur WYSIWYG - Choisissez ici lequel des éditeurs de texte installés vous voulez utiliser pour créer ou modifier vos articles.

Longueur des listes - Indiquez une longueur par défaut pour les listes affichées dans la partie Administration du site ('Administration'). Veuillez noter que si vous avez déjà vu une page comportant une liste, modifier ce réglage dans la Configuration générale n'aura pas d'effet visible immédiatement. Cette valeur ne sera effectivement prise en compte comme paramètre par défaut qu'à partide votre prochaine connexion.

Icône Sites favoris - Cette zone permet de choisir (pour certains navigateurs) l'icône qui apparait dans la barre de titre à côté du nom et/ou de l'adresse du site. Par défaut, il s'agit du logo Elxis. Si vous voulez utiliser une autre icône, vous devez la télécharger sur votre serveur pour remplacer celle de Elxis. Tapez ensuite l'adresse de cette nouvelle icône dans la zone de saisie.

 

Onglet Lieu

Langue partie public - Choisissez la langue utilisée par défaut sur votre site. Si ce n'est pas l'Anglais, vous devez installer les fichiers de langue correspondant en utilisant le menu Installation, avant de pouvoir utiliser cette langue. Vous pouvez installer ou sélectionner d'autres langues en la sélectionnant.

Langue partie administration - Choisissez la langue utilisée par défaut sur votre site. Si ce n'est pas l'Anglais, vous devez installer les fichiers de langue correspondant en utilisant le menu Installation, avant de pouvoir utiliser cette langue. Durant la session, vous pouvez changer la langue de bse en cliquant sur le bouton en haut à droite.

Décalage horaire - Cette option fixe le décalage horaire (en heures) entre l'heure du lieu où se trouve votre serveur et l'heure GMT (Greenwich Mean Time - Royaume-Uni). Choisissez ce décalage dans la liste déroulante proposée.

Paramètre du pays - Vous pouvez utiliser cette option pour sélectionner un système locale. Nous vous recommandons de mettre à -Auto Sélection-, comme cela Elxis utilisera ce qu'il y a de mieux pour vous. Veuillez noter que Windows ne supporte pas le code UTF-8.

Vérification Locale - Apercevez comment les dates seront affichés et basées sur la localisation. Veuillez noter que Windows ne supporte pas le code UTF-8.

 

Onglet Contenu

Titres liés - Choisissez 'Oui' si vous voulez que le titre des articles deviennent des liens hypertextes ("cliquables"). Si vous choisissez 'Non', un lien 'Lire la suite...' sera affiché après le texte d'introduction de vos articles.

Lien Lire la suite - Choisissez 'Voir' pour afficher le texte 'Lire la suite...' à la fin de votre texte d'introduction.

Evaluation par vote - Cette fonction permet d'activer le vote par les utilisateurs sur les articles. Choisissez 'Oui', et une rangée d'étoiles sera affichée sur chaque articles, ainsi qu'une zone de saisie permettant de proposer votre propre vote. Note: la zone de saisie des votes n'est pas affichée pour les articles situés dans listes de type 'Blog'. De plus, la rangée d'étoiles n'est affichée que pour les articles ayant un texte d'introduction.

Nom de l'auteur - Choisissez 'Voir' pour que le nom de l'auteur d'un article soit affiché en même temps que l'article lui-même.

Date et heure de création - Choisissez 'Voir' pour que la date et l'heure de création d'un article soit affichées en même temps que l'article lui-même.

Date et heure de modification - Choisissez 'Voir' que la date et l'heure de la dernière modification d'un article soit affichées en même temps que l'article lui-même.

Clics - Choisissez le bouton 'Voir' pour afficher pour chaque article le nombre de clics dont il a fait l'objet.

Icône RTF - Choisissez le bouton 'Voir' pour afficher l'icône RTF. L' icône RTF est utilisé pour convertir et télécharger un article au format RTF (Rich Text Format).

Icône PDF - Choisissez le bouton 'Voir' pour afficher l'icône PDF lors de l'affichage des articles dans l'interface utilisateur. NOTE: Cela ne s'affichera pas si votre répertoire 'Média' n'est pas accessible en écriture.

Icône Imprimer - Choisissez le bouton 'Voir' pour afficher l'icône permettant d'obtenir une version imprimable d'un article, lors de sa visualisation dans l'interface utilisateur.

Icône E-mail - Choisissez le bouton 'Voir' pour afficher l'icône Envoyer l'article à un ami, lors de la visualisation d'un article dans l'interface utilsateur.

Icônes - Choisissez si vous voulez afficher les icônes ou un texte simple pour les options RTF, PDF, Print et Email.

Publications multi-pages - Choisissez le bouton 'Voir' pour afficher les zones de navigation dans la table des matières des articles multi-pages. Pour créer des sauts de pages dans vos articles, vous devez insérer un joombot 'Page break', {mospagebreak} au moyen d'un éditeur, à l'endroit où vous souhaitez que le saut de page se produise.

Bouton Retour - Choisissez le bouton 'Voir' pour que le bouton de navigation 'Retour' soit affiché sur chaque page, afin de faciliter la navigation dans votre site à vos visiteurs.

Eléments de navigation - Choissisez le bouton 'Voir' pour publier des liens 'Précédent' et 'Suivant' lors de la visualisation d'articles dans une catégorie, ou pour changer de page à l'intérieur d'un même article.

 

Onlget Base de données

Attention: Les réglages suivants sont trés importants. Si quelqu'un d'autre a installé votre site Elxis pour vous, il serait plus prudent de ne pas les modifier.
NE PAS CHANGER CES ELEMENTS SI VOUS NE MAITRISEZ PAS LE SUJET !!

Type de Base de Données - Changez le type de base qu'utilise Elxis.

Nom de l'hôte - La plupart du temps, vous pouvez saisir 'localhost', un nom d'hôte typique.

Nom d'utilisateur - Modifiez, si nécessaire, votre nome d'utilisateur pour l'accès à votre base de données de Elxis.

Mot de passe - Modifiez, si nécessaire, le mot de passe pour l'accès à votre base de données de Elxis.

Base de Données - C'est le nom de la base de données utilisée par Elxis.

Préfixe de base de données MySQL - Modifiez, si nécessaire, le préfixe utilisé pour les tables Elxis dans votre base de données. Typiquement exl_" par défaut. Attention: NE MODIFIER QUE SI VOUS AVEZ UNE BASE DE DONNEES DONT LES TABLES COMPORTE LE PREFIXE QUE VOUS SAISISSEZ!!

 

Onglet Serveur

Les trois premiers éléments de cette page sont prédéfinis durant l'installation et ne sont pas modifiables ici.

Chemin absolu - C'est le nom du répertoire sur le serveur que Elxis utilise pour s'orienter dans le système environnant.

Site en ligne - C'est l'URL absolue (adresse internet) pour votre installation de Elxis. Valeur Typique est http://www.monsite.com.

Mot secret - Ceci est un mot unique et spécifique à chaque installation de Elxis. Il est utilisé pour certaines fonctions de sécurité.

Méthode GZIP - Choisissez le bouton 'Oui' pour autoriser votre serveur internet à envoyer des données compressées, s'il est configuré pour les navigateurs de vos visiteurs, cette fonction peut leur procurer un temps de chargement des pages nettement réduit.

Durée de la session - Indiquez le nombre de secondes durant lequel une session d'administration peut rester active avant qu'une déconnexion automatique se produise. Après une telle déconnexion, l'utilisateur devra à nouveau donner son identifiant et son mot de passe pour se reconnecter. Attention: Si vous êtes en train de modifier un article, mais n'avez pas sauvé votre travail, vous serez déconnecté automatiquement.

Rapport d'erreur - Si cette option est activée dans l'onglet Site, alors la liste déroulante située ici permet de choisir le niveau de précision du rapport d'erreur. Les options possibles sont: par défaut du système, Aucun, Simple et Maximum.

  • par défaut du système - Le rapport d'erreur est celui choisi par défaut pour le système, dans le fichier php.ini sur le serveur.
  • Aucun - Le rapport d'erreur est désactivé.
  • Simple - Le rapport d"erreur est activé en version simple, correspondant à l'option: error_reporting (E_ERROR|E_WARNING|E_PARSE)
  • Maximum - Le rapport d'erreur est activé en version maximum, correspondant à l'option: error_reporting (E_ALL)

Emulation Register Globals - Cette option vous permet de mettre register_globals emulation à On ou Off. Il est vivement recommandé de le mettre à Off mais cela peut provoquer des arrêts des composants tiers. Si vous avez des problèmes avec certaines variables et certains éléments ou modules, essayez de mettre cette option à On.

Serveur d'Aide - Vous pouvez utiliser les sources de Elxis pour les fichiers d'aide. Vous pouvez les afficher et les utiliser en cliquant sur l'icône "Aide". Les options sont:

  • Fichiers en Local: Par défaut. Vous utilisez les fichiers sur votre serveur local, livré avec Elxis.
  • Serveur distant de Elxis: Sélectionnez ici si vous voulez utiliser les fichiers sur un autre serveur.
  • Serveur d'aide Officiel de Elxis: Sélectionnez ici si vous souhaitez utiliser les fichiers Officiels de Elxis.

Creation de Fichiers - Les fichiers créés lors de l'installation de composants, modules et joombots héritent des droits d'accès par défaut du serveur sur lequel est hébergé le site. En choisissant "Changer les droits (CHMOD) des nouveaux fichiers", vous pouvez supplanter ces droits par défaut.Utilisez avec précaution.

Création de Répertoires - Tout comme les fichiers, les répertoires héritent des droits par défaut du serveur d'hébergement. Choisissez "Changez les droits (CHMOD) des nouveaux répertoires" pour supplanter ces droits par défaut. Utilisez avec précaution.

 

Onglet Métadonnées

Ces paramètres, affecte la direction les moteurs de recherche pour l'affichage votre site.

Méta description du site - Tapez une description de votre site, utilisée pour faciliter le travail des moteurs de recherche. Vous devriez limiter votre description à une vingtaine de mots environ, en fonction du moteur de recherche que vous visez. Faites court et en relation avec le contenu du site. Vous pouvez inclure certains de vos mots ou phrases-clés également. Comme certains moteurs prennent en compte plus de 20 mots vous pouvez rajouter une ou deux phrases. Faites en sorte malgré tout, que la partie la plus importante de la description soit bien située dans les 20 premiers mots.

Mots-clefs du site - Le nombre optimal de caractères pour l'ensemble des mots clés couramment admis est de 300 à 500. Ne répetez pas de mots trop souvent, et séparez les par une virgule ou une espace. Vous pouvez grouper 2 ou 3 mots ensemble, entre virgule, comme par exemple dans 'Elxis'.

Insérer les méta tags de titre - Insère le Meta tag 'Titre' dans le code source de chaque page.

nsérer les méta tags de l'auteur - Insère le Meta tag 'Author' dans le code source de chaque page.

 

Onglet E-mail

Paramètres Elxis lié à la façon dont les messages sont envoyés.

Serveur de mail - Choisissez quel système d'E-mai vous sohaitez utiliser : La fonction mail de PHP, Sendmail ou un serveur SMTP.

E-mail de - Indiquez quelle sera l'adresse d'expéditeur qui figurera sur les mails envoyés depuis votre site.

Nom - Choisissez quel nom d'expéditeur figurera sur les mails envoyés depuis votre site.

Chemin Sendmail - Indiquez le répertoire d'accès au serveur Sendmail. Sous Linux c'est normalement /usr/sbin/sendmail. Non applicable sur des installations Windows.

Si vous avez choisi un serveur SMTP pour le serveur de mail, soyez sur de vos réglages. Sinon veuillez ignorer cette étape.

SMTP Authentification - Choisissez 'Oui' si une authentification SMTP est nécessaire à votre serveur SMTP.

SMTP Utilisateur - Si une authentification est requise, indiquez ici le nom d'utilisateur.

SMTP Mot de passe - Si une authentification est requise, indiquez ici le mot de passe.

SMTP Hôte - En général il s'agit de localhost, mais vous devriez vérifier auprès de votre hébergeur.

 

Onglet Cache

Paramètres liés à la mémoire cache. Le Cache est utilisée pour améliorer les performances du site.

Cache - Choisissez 'Oui' pour activer la possibilité de mise en cache par Elxis de certains éléments. cela améliorera généralement les performances de votre site.

Répertoire de cache - Indiquez ici le répertoire dans lequel Elxis doit stocker les fichiers du cache. Il doit être accessible en écriture pour que le cache fonctionne. Contactez votre adminstrateur système our webmaster en cas de difficulté.

Durée du cache - Indiquez ici la durée, en secondes, avant qu'un élement stocké dans le cache n'arrive à expiration et soit rafraîchi.

 

Onglet Statistiques

Paramètres liés à la collecte de statistiques.

Statistiques - Choisissez 'Oui' pour autoriser la collecte de statistiques sur les navigateurs et les domaines de vos visiteurs, en fonction des réglages effectués dans l'onglet 'Contenu'. Si vous choisissez 'Non', ces données ne seront pas enregistrées. Si votre site montre un léger 'délai de réponse' lors de la première visite d'un utillisateur, vous devriez choisir 'Non'.

Journalisation du nombre de visites par date - Si fixé à 'Oui', le nombre d'accès aux articles sera enregistré, à la journée. Il n'y a actuellement aucune fonction d'analyse venant en complément de cette simple fonction d'enregistrement. Attention: Cette fonction a pour effet d'accumuler des volumes d'information très élevés au fil du temps. Assurez-vous d'avoir suffisamment d'espace libre sur votre serveur pour les stocker!

Journalisation des recherches - Choisissez 'Oui' pour provoquer l'enregistrement du texte tapés par vos visiteurs dans la zone 'Rechercher' de votre site. Cela peut vous permettre d'affiner la mise au point du site, en vous donnant des indications sur la façon dont les utilisateurs recherchent l'information dont ils ont besoin. L'analyse des données enregistrées est disponible dans l'onglet Site -> Statistiques - > Recherche.

 

Onglet SEO

SEO = Search Engine optimisation: Optimisation pour les moteurs de recherche

URL compatibles avec les moteurs de recherche (SEF) - Cette fonction modifie la façon dont les liens hypertextes sont présentés, et les optimisent de manière à ce que les moteurs de recherche visitent une plus grande part de votre site. Elle n'est accessible QUE pour les sites hébergés par un serveur Apache. Choisissez 'Oui' pour que Elxis crée des URL optimisées plutôt que les URLs normales issues de la base de données. Si vous faites cela, vous devez de plus renommer le fichier 'htaccess.txt' situé à la racine de votre installation Exlis en '.htaccess' (avec un point comme premier caractère).
Options possibles:

  • Non: Les URLS SEF sont désactivées. Elxis donnera des URLS classiques
    Par exemple:
    index.php?option=com_content&task=category§ionid=3&id=18&Itemid=32
  • SEO de Base: Les URLS SEF sont activées mais utilise un format plus traditionnel. Select this if you want to use a 3rd party component.
    Par exemple:
    content/category/3/18/33/
  • SEO Pro: Les URLS SEF sont activés et utilise un format de type professionnel, plus facile à lire . Ceci est le meilleur choix.
    Par exemple:
    faqs/managing_content/editing-content.html

Titres des Pages Dynamiques - Choisissez "Oui" pour inclure un titre de page et titre courant aux éléments de contenus. Encore une fois, cela aidera les moteurs de recherches.

 

Onglet FTP

Activer FTP - Choisissez ici l'activation ou pas du transfert des fichiers par le FTP. L'étape suivant vous demandera les paramètres de connexion uniquement si cette option est mis à "Oui".

FTP Hôte - Nom du serveur FTP de votre site.

FTP Utilisateur - Nom d'utilisateur du compte FTP.

FTP Mot de Passe - Mot de Passe du compte FTP.

FTP Port - Port du serveur FTP. Par défaut la valeur est 21.

FTP Chemin du serveur - Chemin d'accès au serveur FTP pour votre installation de Elxis, i.e. /public_html/elxis

NOTE: Vous devez sauvez vos modifications pour qu'elle soient prises en compte, au moyen de l'icône 'Enregistrer' située dans la barre d'outil.
