Διαχειριστής Module

Modules are used to display information items or interactive features throughout an Elxis Site. They are positioned in places defined in templates, that are called module positions. These positions act as placeholders for one or more modules.

Examples of modules are Menus, Login and Search forms. Simple HTML Modules can be created directly from the Modules Manager. In more complicated cases, they will have to be developed and installed.

Modules Manager lists and manages installed modules.

 

Filter

You can use the Filter field to limit listed modules or search for specific module. You can type a phrase or part of a word and press <Enter>. The table will be updated showning only the modules that have these characters in their title. If you want to show again all modules, leave empty and press <Enter>.

 

Column Filters

You will notice that several columns have an arrow next to there headers. When you click them a filter window popups. You can use this filters to select the modules that you want to be displayed. This feature is addressed to web sites that have a large number of modules. It helps administrators to focus only to modules that each time interest them.

 

Toolbar Icons

Publish and Unpublish: Publishes or unpublishes selected module(s). Only published modules are visible in this web site.

Copy: Copies selected modules, creating duplicates. Copied modules names start with 'Copy of'. Of course you can change their name.

New: Creates a new module of type mod_user.

Edit: Edit the selected module.

Delete: Deletes the selected module(s).

Help: This Help Screen.

 

Columns

Module Name: Module's name is used as module's title.

Published: Shows if the module is published or not. Click to change state. Acts as a toggle switch.

Reorder: You can change the order of modules, by using the arrows. That will be the order modules that they will be displayed in Front-End. You can also change the order by using the 'Order' boxes, described next.

Order: Shows the current order of modules. You can change it, by editing the order numbers and then clicking the "Save Order" icon next to column header.

Access: This is the user group that is allowed to view this module. Click on it to change it. A popup window will appear.

Position: Shows in which module position it will be displayed.

Pages: It will be empty if the module is shown for all pages. It will show 'Varies' if it will be shown only in certain pages. It will show 'None' if it will not be shown.

ID: Module's ID number.

Type: Shows module's type. A common type is 'mod_mainmenu' which is the "type" of modules used to display menus.

Languages: Shows for which site languages this module is available. It will be shown only if users are viewing this web site in one of the listed languages. If it is set to 'All', then it will be visible for all languages.

 

Footer

At the bottom of the page there is the footer.

Navigation - If the number of modules is larger than the number of modules displayed per page, then navigation controls are activated. You can use them to switch pages.

Display # - Use this control to change the number of displayed modules per page. The default value is controlled from 'List Length' parameter defined in Global Configuration.
